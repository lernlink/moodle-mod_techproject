Ce sélecteur permet de relier les besoins aux spécifications, dans le sens suivant :

"La spécification X est une description de la solution technique et/ou fonctionnelle qui réalise le besoin Y"

La sélection est multiple : une spécification peut réaliser plusieurs besoins simultanément. Par exemple, la spécification globale d'une "interface graphique" pour gérer une entité pourrait remplir simultanément les 3 besoins exprimés par :

  1. pouvoir ajouter une entité.
  2. pouvoir modifier une entité.
  3. pouvoir supprimer une entité.

A l'inverse, plusieurs spécifications peuvent concourir à la réalisation d'un besoin exprimé de façon très générale. Il est donc possible de retrouver dans la liste des besoins disponibles des entrées qui ont déjà été associées à d'autres spécifications.
