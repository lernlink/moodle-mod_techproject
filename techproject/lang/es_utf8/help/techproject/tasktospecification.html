Une tâche peut être désignée comme réalisant certaines spécifications. Une tâche peut en réalisr plusieurs, si sa définition est assez générale (par exemple : "développer tel module" est une définition très générale qui peut accomplir un ensemble de descriptions détaillées du module en question). A l'inverse, plusieurs tâches peuvent être nécessaires pour accomplir complètement une spécification unique.

Cette liste permet d'associer la tâche courante (actuellement éditée) et les spécifications qu'elle est sensée accomplir. La sélection est multiple.

En effectuant ces associations, vous permettez à certains indicateurs de se propager à travers les entités. Il sera par exemple possible de visualiser le "degré de réalisation" de certaines spécifications à travers le pourcentage de réalisation des tâches qui les réalisent.
